## Integration: ProdMCP + MCPcrunch

The most common way to use MCPcrunch is as an audit layer for your ProdMCP applications. This ensures that the specifications generated by your code are secure and valid before you share them with AI agents.

### Example 1: Export and Audit (Dynamic Audit)

This example shows how to build a simple ProdMCP app, export its specification, and immediately audit it with MCPcrunch in a single script.

```python
import json
from prodmcp import ProdMCP
from pydantic import BaseModel
from mcpcrunch import MCPcrunch

# 1. Initialize ProdMCP App
app = ProdMCP("SecurityTool", version="1.0.0")

class UserInput(BaseModel):
    user_id: str

@app.tool()
def get_user_data(user_id: str):
    """Fetch secure user information."""
    return {"id": user_id, "status": "active"}

# 2. Export the OpenMCP Specification
# ProdMCP automatically generates the spec based on your decorators
spec_dict = app.export_openmcp()

# 3. Initialize MCPcrunch and Audit
# Use the official OpenMCP schema for validation
crunch = MCPcrunch(schema_path="schema.json")
report = crunch.audit(spec_dict)

# 4. Handle results
print(f"Audit Score: {report.overall_score}/100")
if report.overall_score < 80:
    print("⚠️ Security score too low! Review the reported issues below:")
    for issue in report.deterministic.issues:
        print(f"[{issue.severity}] {issue.rule_id}: {issue.message}")
else:
    print("✅ Specification is secure and production-ready.")
```

### Example 2: Pre-deployment Pipeline (Static Audit)

If you generate your specification as a JSON file during your build process, you can use MCPcrunch as a gatekeeper in your deployment pipeline.

```python
import sys
import json
from mcpcrunch import MCPcrunch

def run_security_check(spec_file_path: str):
    with open(spec_file_path) as f:
        spec_data = json.load(f)

    crunch = MCPcrunch(schema_path="schema.json")
    report = crunch.audit(spec_data)

    if report.overall_score < 90:
        print(f"❌ Security check failed! Score: {report.overall_score}")
        # Fail the build if the score is below the threshold
        sys.exit(1)

    print(f"✅ Security check passed with score: {report.overall_score}")

if __name__ == "__main__":
    run_security_check("openmcp.json")
```

## Reverse Integration: Using MCPcrunch in ProdMCP

### Example 3: Automated Testing with Pytest

You can use MCPcrunch within your ProdMCP project's test suite to ensure that any changes to your tools or schemas don't introduce security regressions.

```python
import pytest
from prodmcp import ProdMCP
from mcpcrunch import MCPcrunch

@pytest.fixture
def my_app():
    app = ProdMCP("TestServer")
    # ... add tools ...
    return app

def test_specification_security(my_app):
    spec = my_app.export_openmcp()
    crunch = MCPcrunch(schema_path="schema.json")
    report = crunch.audit(spec)
    
    # Assert that the specification is robust
    assert report.overall_score >= 80, f"Specification security score {report.overall_score} is too low!"
    assert len(report.deterministic.issues) == 0, "No deterministic issues should be present."
```

## Advanced: Guarding against Prompt Injection

When using LLM-powered auditing, you can specifically check for vulnerabilities in your tool and prompt descriptions.

```python
from mcpcrunch import MCPcrunch, GeminiProvider

# Initialize with Gemini for semantic analysis
llm = GeminiProvider(api_key="your-api-key")
crunch = MCPcrunch(schema_path="schema.json", llm=llm)

report = crunch.audit(spec_data)

# Specifically look for Adversarial (ADV) issues
adversarial_issues = [
    i for i in report.semantic.issues 
    if i.rule_id.startswith("OMCP-ADV-")
]

if adversarial_issues:
    print("🚨 Potential Prompt Injection or Sensitive Leakage detected!")
```
